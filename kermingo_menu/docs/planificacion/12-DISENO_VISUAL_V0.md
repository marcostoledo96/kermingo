# 12 — Diseño visual basado en v0

## Fuente visual obligatoria

```txt
/home/marcos/Escritorio/Kermingo/kermingo_menu/diseno-de-landing-kermingo
```

Esta carpeta es la base visual principal del frontend.

## Regla

Cualquier cambio de UI debe partir del diseño existente en `diseno-de-landing-kermingo/`.

No usar diseños genéricos de IA.  
No rehacer desde cero.  
No transformar la landing en dashboard.

## ADN visual actual

Mantener:

- Fondo celeste claro.
- Azul oscuro como color principal.
- Celeste argentino.
- Amarillo/dorado para CTAs.
- Cards blancas con bordes redondeados.
- Sombras suaves.
- Franja argentina.
- Hero fuerte.
- Detalles de Día de la Bandera.
- Mundial/Argentina.
- Scout sutil.
- Mobile-first.
- Actividades compactas.
- Botones grandes.

## Pantallas públicas

Deben sentirse:

- festivas
- familiares
- claras
- mobile-first
- rápidas de usar

## Pantallas admin

Deben sentirse:

- operativas
- claras
- menos decorativas
- enfocadas en resolver el evento real

## Dashboard admin

Marcos indicó que es la pantalla que menos lo convence.

Al mejorarla:

- reducir cards genéricas
- priorizar pedidos pendientes
- pagos a validar
- stock bajo
- accesos rápidos a caja/cocina
- total recaudado
- estado de tienda
- últimos pedidos reales

## Pantallas faltantes

Crear siguiendo el mismo ADN visual:

```txt
/admin/comprobantes
/admin/reportes
/admin/configuracion
```
